# 发给老婆的使用方案

## 方案 A：你们在同一个 Wi‑Fi 下，马上可用

电脑这边保持服务运行，然后把这个链接发给她：

```text
http://10.4.134.154:8766/cooking-randomizer.html
```

适合在家里同一个 Wi‑Fi 下使用。电脑关机、换 Wi‑Fi、服务关闭后，这个链接就打不开。

## 方案 B：做成真正可长期使用的公开链接

这个程序是纯静态网页，最适合部署到 GitHub Pages、Netlify、Vercel 或 Cloudflare Pages。部署后你会得到一个 `https://...` 链接，直接发给她就能打开，不要求同一个 Wi‑Fi。

需要上传/部署的内容是整个 `outputs` 目录，至少包含：

```text
cooking-randomizer.html
assets/cartoon-theme.jpg
```

## 数据记录说明

食材、菜谱照片、做饭记录、当天随机历史都保存在“当前手机浏览器”的本地存储里。

也就是说：

- 她用自己手机打开后，会建立她手机上的记录。
- 之后她继续用同一个浏览器打开，记录不会重新开始。
- 如果她清理浏览器数据、换手机、换浏览器，记录会重新开始。

## 今日随机规则

程序已经加入“当天前三次尽量不重复”的规则：

- 同一天第 1、2、3 次随机，会优先避开当天已经随机过的菜。
- 如果可选菜少于 3 道，会在可选范围内尽量不重复。
- 第 4 次以后允许重复，避免菜谱池太小时一直随机不出来。
